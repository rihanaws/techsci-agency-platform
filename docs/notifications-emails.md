# Notifications & Transactional Emails

This document details the system's dispatch mechanism for customer transactional emails and real-time Telegram alerts.

---

## 📢 Telegram Owner Alerts

To keep the platform's owner updated on checkouts, form submissions, report compilation states, and server exceptions, the platform utilizes a Telegram bot notification system.

The code is located in [lib/telegram/notify.ts](file:///Users/rihan/all-coding-project/techsci-agency-platform/lib/telegram/notify.ts).

### API Configuration
The notifier communicates directly with the Telegram Bot HTTP API using two environment variables:
- `TELEGRAM_BOT_TOKEN`: The unique token generated by BotFather.
- `TELEGRAM_CHAT_ID`: The chat/channel ID where notifications should be posted.

### Notification Flow
The helper sends HTML-formatted messages synchronously or asynchronously:
```typescript
export async function notifyOwner(message: string): Promise<boolean> {
  const token = process.env.TELEGRAM_BOT_TOKEN
  const chatId = process.env.TELEGRAM_CHAT_ID

  if (!token || !chatId) {
    console.warn('[telegram/notify] Missing configuration — skipping notification')
    return false
  }

  try {
    const url = `https://api.telegram.org/bot${token}/sendMessage`
    const res = await fetch(url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        chat_id: chatId,
        text: message,
        parse_mode: 'HTML',
      }),
    })
    return res.ok
  } catch (err) {
    console.error('[telegram/notify] Failed to send Telegram alert', err)
    return false
  }
}
```

## ✉️ Resend Transactional Emails

The platform delivers professional transactional HTML emails using the **Resend API**.

All email dispatches are configured in [lib/resend/emails.ts](file:///Users/rihan/all-coding-project/techsci-agency-platform/lib/resend/emails.ts) and utilize the client wrapper in [lib/resend/client.ts](file:///Users/rihan/all-coding-project/techsci-agency-platform/lib/resend/client.ts).

### 📧 Supported Email Dispatches

#### 1. ZIP File Delivery
Triggered on a successful one-time ZIP product checkout (`ai-agent-pack` or `make-scenario-pack`).
- Generates 48-hour secure presigned download links for the files mapping to the product slug.
- Sends an email listing the links with a stylized call-to-action button.

#### 2. PDF Report Delivery
Triggered after a customer submits an intake form and the system completes generating the report.
- Contains a call-to-action button pointing directly to the secure 48-hour presigned URL of the generated PDF report.

#### 3. Onboarding Welcome Email
Triggered when a subscription is activated (e.g. for `community`).
- Welcomes the customer to the platform and outlines their onboarding steps.
- Explains how to access their purchase features.
